sql语句执行顺序

Oracle

在 Oracle 数据库上，每个SELECT语句必须指定一个表来选择FROM 。即使UNION SELECT攻击不从表中查询，仍然需要包含FROM关键字，后跟有效的表名称。

而Oracle 上有一个名为dual内置表，可以借助其完成

union

判断原select列数

union判断列数,一般有两个办法

order by n

对select语句的第n列结果集进行排序,那么当n（order by n）超过了原本的select语句的列的数量，那么就会报错，解释其n的数字超过了select语句本身的列数

union select null,[null….]

在union语句里面，是要求前后的select语句的列数和类型是一致的，因此可以添加不同数量的null来进行列数的判断，当null的数量超过了前面的select语句的列数，就会导致报错

oracle还需要from dual

必须多列处理

如果遇到了，攻击者需要返回多列的数据，而该sql注入只查询了一列数据的时候该如何

这个时候就需要把多列的数据给串联起来，在不同的数据中，串联的方法是不一样的。

不过一般只有字符串能这么做

Oracle ‘col1’\ \ ‘col2’
Microsoft ‘col1’+’col2’
PostgreSQL ‘col1’\ \ ‘col2’
MySQL 'col1''col2'
concat(col1,col2)
