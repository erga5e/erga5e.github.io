DOM

详细可看理解DOM - 前端开发者学堂 (fedev.cn) - 前端开发社区

DOM(Document Object Model)直译过来是文档对象模型

这个概念可以帮助理解一些XSS进阶技巧

一个常规的WEB网站是由一个html文档组成,当我们访问一个网页时浏览器负责解释html和CSS,同时浏览器还会创建一个文档对象模型,也就是DOM

DOM是整个html的抽象表示,由称为节点的对象树组成,可以被方便地被javascript访问

在DOM中，所有的项都定义为节点,节点的类型有很多种,可以参考Node: nodeType property - Web APIs | MDN (mozilla.org)

元素

一个标准的 HTML 元素包含以下三部分：

  • 起始标签（Start Tag）：<标签名>，定义元素的开始，并告诉浏览器如何处理该内容。
  • 内容（Content）：位于起始标签和结束标签之间的内容，通常是文本或其他 HTML 元素。
  • 结束标签（End Tag）：</标签名>，表示该元素的结束。

有些 HTML 元素是自闭合元素，它们不需要结束标签，因为它们没有内容。例如，<img> 用于插入图片，它是一个自闭合元素。

htmlspecialchars() 将特殊的 HTML 字符（如 <, >, &, " 等）转换为它们的 HTML 实体，从而防止用户注入的 HTML 或 JavaScript 代码被浏览器执行。

例如<script>alert('XSS')</script>会被htmlspecialchars()会将其转义为 &lt;script&gt;alert(&#039;XSS&#039;)&lt;/script&gt;

DOM

window.location 是一个 JavaScript 对象，包含当前页面 URL 的信息。window.location.hash 返回 URL 中的片段标识符，也就是 # 后面的内容。

slice(1) 是字符串的 JavaScript 方法，表示从索引 1 位置开始截取字符串。因为 window.location.hash 的值以 # 开头，使用 slice(1) 去掉了 #，只保留后面的部分。

decodeURIComponent() 是 JavaScript 的全局函数，用于解码通过 encodeURIComponent() 编码的 URI 组件。它会将 URL 中的编码字符（如 %20 代表空格）还原为普通字符。

$(window).on('hashchange', function(){ ... }):

  • $(window)：选择当前窗口对象。
  • .on('hashchange', function(){ ... })：绑定了一个 hashchange 事件监听器，当 URL 中的 hash 部分（# 后面的部分）发生变化时，触发这个事件，并执行其中的回调函数。
  • hashchange 事件：当用户修改浏览器的 URL 中 # 后的部分时（例如点击带有锚点的链接，或手动改变 URL 中的 hash 值），这个事件会被触发

网页会检测url中给的hash变化,一旦变化就会调用该函数
