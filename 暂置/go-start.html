go关键字

go关键字用于启动一个新的并发执行的函数（称为Go协程）。通过go关键字，可以在程序中创建新的协程来并发执行指定的函数。go关键字后面跟着要执行的函数调用。

协程goroutine和线程thread在概念上有一些重要的区别

  • 程由操作系统内核进行调度和管理，内核线程（Kernel-level thread）是操作系统内核的一部分，因此线程的调度和切换是由操作系统来完成的。
  • 协程则由运行时（例如Go语言的运行时）进行调度和管理，它们是由编程语言的运行时库实现的，因此调度和切换是在用户空间完成的，通常比线程更轻量级。

结构体标签

结构体标签（struct tag）是Go语言中用于为结构体字段附加额外信息的一种机制。结构体标签通常是写在字段声明的后面，用反引号包裹起来，例如 json:"name"。

结构体标签的主要作用是为结构体字段提供元数据，通常用于实现字段的序列化、反序列化、验证等功能。在实际应用中，结构体标签经常用于标记字段，以便于其他库或工具根据标签来进行特定操作。

以下是结构体标签的一些常见用法和示例：

  • 序列化和反序列化：结构体标签通常用于指定字段在序列化为JSON、XML等格式时的名称。

在这个例子中，json:"name" 和 json:"email" 是用于指定字段在JSON格式中的名称。当使用json.Marshal函数将 User 结构体编码为JSON时，字段 Name 和 Email 将会被编码为 "name" 和 "email"。

  • 验证和约束：结构体标签也可以用于指定字段的验证规则或约束条件。

在这个例子中，validate:"required" 指定了 Name 字段是必填项，validate:"gte=0,lte=130" 指定了 Age 字段的值必须在0到130之间，validate:"email" 指定了 Email 字段必须是有效的电子邮件地址。

  • ORM映射：在使用ORM（对象关系映射）库时，结构体标签通常用于指定字段与数据库表的映射关系。

在这个例子中，gorm:"primary_key" 指定了 ID 字段是主键，gorm:"column:name" 指定了 Name 字段在数据库表中的列名为 name。

select 关键字是 Go 语言中用于处理并发操作的一种机制，它可以用来同时等待多个通信操作，并在其中任意一个操作完成时进行响应。select 语句通常与 chan 类型一起使用，用于处理通道之间的并发通信。

下面是 select 关键字的基本语法：

select 语句会阻塞，直到其中一个 case 条件成立（通道操作可完成）。当多个 case 条件同时成立时，select 会随机选择其中一个执行。如果没有任何 case 条件成立并且存在 default 分支，则会执行 default 分支（如果没有 default 分支，则 select 语句会一直阻塞直到某个 case 条件成立）。

以下是 select 关键字的一些常见用法：

  • 等待通道操作：select 可以用于同时等待多个通道的读写操作。
  • 超时处理：select 结合 time.After 可以用于实现超时机制，例如在一定时间内等待通道操作完成，否则执行超时处理。
  • 多路复用：select 可以用于同时等待多个通信操作完成，然后响应任意一个操作的完成。

总的来说，select 关键字是 Go 语言中用于处理并发通信的重要机制，它能够有效地处理多个通道之间的并发操作，并在通信操作完成时进行响应，使得 Go 语言中的并发编程变得更加简洁和灵活。

函数调用栈约定

Go 1.17 版本之前，采用基于栈的调用约定，即函数的参数与返回值都通过栈来传递，这种方式的优点是实现简单，无需考虑不同架构下cpu寄存器的差异，更容易实现跨平台。然而由于内存的访问时延远高于寄存器，导致 Golang 的函数调用性能要低于 C/C++ 等使用寄存器传递参数的编程语言。

Golang 在 1.17 版本中引入了基于寄存器的函数调用约定，本文将简要介绍该调用约定，并通过具体的实验来验证并加深理解。

栈帧结构如下：栈帧由高地址向低地址增长，依次是上一个函数的程序计数器 (Program Counter, PC)、上一个函数的基址寄存器 (Base Pointer, BP)、函数的局部变量、调用函数的返回值、调用函数的参数。

寄存器约定 (amd64)：

  1. 用于整型参数和返回值的通用寄存器：RAX、RBX、RCX、RDI、RSI、R8、R9、R10、R11
  2. 用于浮点参数和返回值的浮点寄存器：X0~X14
  3. 其他是用于特殊目的的寄存器：
寄存器 名称含义 值含义
RSP 栈顶指针 (Stack pointer) 固定
RBP 栈帧指针（基址）(Frame pointer) 固定
RDX 闭包上下文指针 临时
R12 无 临时
R13 无 临时
R14 当前 go routine 临时
R15 GOT reference temporary 动态链接时固定
X15 零值寄存器 固定为0

函数的调用者负责保存寄存器，被调用者无需保存寄存器，因此可以会覆盖任何没有固定含义的寄存器，包括参数寄存器。

在go1.22.1环境下编译完成后,ida打开分析一下

  • 与c/c++等使用rbp进行局部变量寻址不同,go采用rsp进行局部变量寻址

函数启动过程

推荐文章Go 程序是怎么运行起来的？（Go1.21.0） - 知乎 (zhihu.com)

go编译出来的可执行文件基本都是静态链接

  1. 命令行参数复制：读取命令行参数，复制到 argc 和 argv。
  2. 初始化 g0 栈：g0 是运行时系统的一个特殊的 goroutine，它在程序启动时被创建，用于执行系统调用和协程调度。
  3. runtime.check 运行时检查：
    1. 类型长度
    2. 指针操作
    3. 结构体字段偏移量
    4. CAS
    5. atomic 操作
    6. 栈大小是否为 2 的幂次。
  4. runtime.args 参数初始化：将 argc 和 argv 的参数赋值到 Go 的变量中。
  5. runtime.osinit 初始化操作系统特点的设置：主要是判断系统字长和 CPU 核数。
  6. runtime.schedinit 初始化调度器：
    1. 锁初始化
    2. 竞态检测器初始化
    3. 调度器设置，设置调度器可以管理的最大线程（M）数目
    4. 系统初始化，初始化内存管理、CPU 设置、算法等，这些都是调度器正常工作的基础
    5. 设置当前 M 的信号掩码
    6. 解析程序参数和环境变量
    7. 垃圾收集器初始化
    8. 设置 process 的数量
  7. runtime.newproc 创建主协程 g0 并将其放入队列中等待执行。
  8. runtime. mstart 启动调度器：初始化 m0，并调度 g0 去执行 runtime.main。
  9. runtime.main 程序真正入口：
    1. runtime.init
    2. 启动 gc
    3. 执行用户包 init
    4. 执行用户函数 main.main

由于go是静态编译,栈溢出之后就是常规的ret2syscall,此外由于go的运行时模拟的堆栈的地址是固定的,也就是说不存在随机化,那么像/bin/sh字符串可以直接由自己写入,然后调试出地址并直接填到exp中
