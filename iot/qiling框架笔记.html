qiling框架笔记

qiling框架是一个基于python的二进制分析模拟和虚拟化框架

其可以用动态分析和仿真运行不同操作系统,处理器和体系结构下的二进制文件

并且提供了易于使用的api接口和插件系统,方便用户进行二进制分析以及漏洞挖掘等工作

qiling是基于unicorn的,qiling框架的创始人本身就是一个iot hacker,其创建qiling框架的目的之一就是解决研究iot时遇到的各种问题,因此与unicorn相比,其更适合iot初学者,比如我orz

这是qiling的官方文档Qiling Framework Documentation

基础使用

qiling没有提供中文文档,为了方便使用记录并实践一些文档中api使用方法

Qiling()

使用qiling框架必需包含的一个环节就是初始化一个Qiling对象

二进制模拟

  1. argv
    • Type: Sequence[str]
    • Description: 一个命令行参数序列，用于仿真。在仿真过程中，这些参数将被传递给仿真的程序，模拟其在真实环境中被执行时所接收的命令行参数。
  2. rootfs
    • Type: str
    • Description: 仿真文件系统的根目录。仿真的程序访问的所有路径都将基于这个目录。这意味着在仿真过程中，所有的文件访问操作都将相对于此根目录进行，从而模拟一个完整的文件系统环境。
  3. env (optional)
    • Type: MutableMapping[AnyStr, AnyStr]
    • Description: 一个可选的环境变量字典。这些环境变量将可用于仿真的程序。通过这个字典，可以为仿真程序设置特定的环境变量，从而模拟其在不同环境下的行为。

shellcode模拟

  1. code
    • Type: bytes
    • Description: 要仿真的shellcode字节码。这取代了argv参数，即直接提供要执行的shellcode而不是命令行参数。
  2. rootfs (optional)
    • Type: str
    • Description: 仿真文件系统的根目录。这个选项在上面的表格中也有详细描述，仿真的程序访问的所有路径都基于这个目录。
  3. ostype
    • Type: str 或 QL_OS
    • Description: 设置目标操作系统（不区分大小写）。可选值包括：’Linux’, ‘FreeBSD’, ‘MacOS’, ‘Windows’, ‘UEFI’, ‘DOS’, ‘EVM’ 或 ‘QNX’。
  4. archtype
    • Type: str 或 QL_ARCH
    • Description: 设置目标架构（不区分大小写）。可选值包括：’a8086’, ‘x86’, ‘x8664’, ‘ARM’, ‘ARM64’, ‘MIPS’, ‘EVM’, ‘Cortex_M’, ‘RISCV’ 或 ‘RISCV64’。
  5. endian (optional)
    • Type: bool
    • Description: 指定架构的字节序，仅对ARM和MIPS架构相关。
  6. thumb (optional)
    • Type: bool
    • Description: 指定ARM的Thumb模式，仅对ARM架构相关。

通用选项

  1. verbose (optional)
    • Type: QL_VERBOSE
    • Description: 设置Qiling的日志详细级别（默认值为 QL_VERBOSE.DEFAULT）。
  2. profile (optional)
    • Type: str
    • Description: 指定包含额外设置的配置文件路径。更多详情见profile部分。
  3. console (optional)
    • Type: bool
    • Description: 当设置为False时，完全禁用Qiling日志记录。相当于设置 verbose=QL_VERBOSE.DISABLED。
  4. multithread (optional)
    • Type: bool
    • Description: 指定目标程序是否应作为多线程程序仿真。
  5. libcache (optional)
    • Type: bool
    • Description: 指定是否应从缓存加载库。这可以在后续运行时节省库解析和重定位的时间。目前仅适用于Windows。
